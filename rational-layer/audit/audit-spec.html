Audit Engine

Audit‑Specs

Formale Regeln der Genesis‑Engine — unveränderlich, deterministisch, audit‑stabil.

Was sind Audit‑Specs?

Audit‑Specs sind die maschinenlesbaren, unveränderlichen Regeln der Genesis‑Engine. Sie definieren, welche Bedingungen immer erfüllt sein müssen, damit das System als korrekt gilt.

Sie bilden die Grundlage für:

  • Audit‑Flows
  • Audit‑Protokolle
  • Ritual‑Validierung
  • Payout‑Integrität
  • Operator‑Kohärenz

Spec 1 — Payout‑Kohärenz

RULE: NO_PAYOUT_WITHOUT_CLOSURE IF payout.exists THEN closure.exists AND closure.references(payout)

Jeder Payout muss durch ein gültiges Closure‑Event begründet sein.

Spec 2 — Closure‑Kohärenz

RULE: NO_CLOSURE_WITHOUT_ECHO IF closure.exists THEN echo.exists AND echo.references(closure)

Jedes Closure‑Event muss ein Echo erzeugen.

Spec 3 — Operator‑Validität

RULE: NO_ACTION_WITHOUT_ROLE IF operator.action THEN operator.role.is_valid AND operator.role.allows(action)

Keine Operator‑Aktion ohne gültige Rolle und Berechtigung.

Spec 4 — Genesis‑Unveränderlichkeit

RULE: GENESIS_IMMUTABLE IF state.height > 0 THEN genesis_state MUST NOT CHANGE

Die Genesis ist unveränderlich nach dem Start der Chain.

Spec 5 — Drift‑Integrität

RULE: DRIFT_REQUIRES_ECHO IF drift_signal.exists THEN echo.exists AND echo.references(drift_signal)

Drift‑Signale müssen durch Echo‑Einträge begründet sein.

Spec 6 — Ritual‑Kohärenz

RULE: RITUAL_TRANSITION_VALID IF ritual.transition THEN transition.follows(ritual.rules) AND operator.role.allows(transition)

Jeder Ritual‑Übergang muss den formalen Regeln entsprechen.

Spec 7 — Payout‑Sequenz‑Integrität

RULE: PAYOUT_SEQUENCE_ORDERED IF payout.sequence THEN payout.sequence.is_monotonic AND payout.references_previous_closure

Payout‑Sequenzen müssen geordnet und lückenlos sein.

Spec 8 — Echo‑Konsistenz

RULE: ECHO_CONSISTENT IF echo.entry THEN echo.entry.references_valid_state AND echo.entry.is_append_only

Echo‑Einträge müssen konsistent und append‑only sein.
