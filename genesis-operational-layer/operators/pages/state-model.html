Operator State Model

Deterministisches Zustandsmodell eines Operators innerhalb der Genesis‑Engine.

Beschreibung

Das Operator‑State‑Model definiert alle Felder, die den Zustand eines Operators beschreiben. Der State ist vollständig deterministisch, audit‑fähig und wird ausschließlich aus der Audit‑Kette rekonstruiert. Es existiert kein „mutierender“ Zustand — nur abgeleitete Zustände aus Events.

State‑Felder

Feld Typ Beschreibung
operatorId string Deterministische Identität des Operators.
publicKey string Ed25519‑Public‑Key des Operators.
role string Funktionale Rolle des Operators.
permissions array[string] Granulare Berechtigungen des Operators.
seal string Signiertes Identitäts‑Artefakt.
active boolean Aktivierungsstatus des Operators.
auditChainHash string Letzter Hash der Operator‑Audit‑Kette.

State‑Regeln

  • Der State ist niemals mutierend — nur abgeleitet.
  • Jede Änderung erfolgt über ein Event.
  • Der State ist vollständig reproduzierbar.
  • Der State ist append‑only durch die Audit‑Kette.
  • Der State ist deterministisch — gleiche Events → gleicher State.
