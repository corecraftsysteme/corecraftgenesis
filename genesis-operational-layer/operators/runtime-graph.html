Operators — Runtime Graph Viewer

Deterministischer Ausführungsgraph für Genesis‑Operatoren.

Runtime Graph — Übersicht

Der Runtime‑Graph zeigt die vollständige Ausführungslogik eines Operators: Zustände, Übergänge, Trigger‑Events und auditierbare Operationen.

Jeder Operator folgt einem deterministischen Graphen, der in Audit‑Logs, Manifeste und WORM‑Archive eingebettet ist.

Graph‑Struktur

Init
 └── enable → Active
Active
 ├── pause → Paused
 ├── error → Error
 └── retire → Retired
Paused
 └── resume → Active
Error
 └── recover → Active
Retired
 └── (terminal)

Jeder Übergang erzeugt ein auditierbares Event im Operator‑Audit‑Log.

Trigger‑Events

  • operator.enable — aktiviert Operator
  • operator.pause — pausiert Operator
  • operator.resume — setzt Operator fort
  • operator.retire — deaktiviert Operator dauerhaft
  • operator.error — Fehlerzustand
  • operator.recover — Fehlerbehebung

Audit‑Integration

  • Alle Runtime‑Transitions werden in operator-audit.json protokolliert.
  • Batch‑Jobs erzeugen tägliche audit-log-YYYY-MM-DD.json.
  • S3‑WORM sichert die Unveränderlichkeit.
  • IPFS‑CID verankert die globale Referenz.
  • Genesis‑Manifeste referenzieren alle Runtime‑Graph‑Artefakte.
