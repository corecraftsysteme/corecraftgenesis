Operator State‑Model

Deterministisches Zustandsmodell eines Operators innerhalb der Genesis‑Engine.

Überblick

Das State‑Model definiert alle gültigen Zustände eines Operators sowie die erlaubten Übergänge. Jeder Übergang ist deterministisch, signiert und Teil der Audit‑Kette.

Zustände

  • CREATED – Operator wurde erzeugt
  • ACTIVE – Operator ist aktiv und autorisiert
  • DISABLED – Operator wurde deaktiviert
  • SEALED – Operator hat ein neues Seal erhalten
  • UPDATED – Rollen oder Permissions wurden geändert

State‑Diagramm

CREATED │ ▼ ACTIVE ────────► UPDATED ────────► ACTIVE │ ▲ │ │ └──────────────► DISABLED

Übergänge

  • operator_created → CREATED
  • role_assigned → UPDATED
  • role_changed → UPDATED
  • permission_granted → UPDATED
  • permission_revoked → UPDATED
  • seal_generated → SEALED
  • operator_disabled → DISABLED

Audit‑Integration

Jeder Übergang erzeugt ein deterministisches Audit‑Event:

  • SHA‑256‑Hash
  • Ketten‑Hash
  • Ed25519‑Signatur
  • Zeitstempel

Der vollständige Operator‑State wird aus der Audit‑Kette rekonstruiert.

Beispiel‑State

{ "operatorId": "OPR-STATE-EXAMPLE-01", "state": "ACTIVE", "role": "RITUAL_OPERATOR", "permissions": [ "canTriggerRituals" ], "active": true }

Zusammenfassung

Das State‑Model definiert die gesamte Lebenslinie eines Operators. Es ist deterministisch, audit‑fähig und Grundlage für Governance und Sicherheit.
